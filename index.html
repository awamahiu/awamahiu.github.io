Atlas Quick Start Guide

About Atlas Viewer
AtlasViewer is a cloud-based data mapping tool that allows users to visualize location-specific data. The idea is that we all have data…lots and lots of data…sometimes too much data…and we need to analyze all of this data and use it to make decisions. The problem is that we can get buried in all this data. It’s especially complicated when the data is geographically dispersed, is changing in real-time, and may be coming from multiple sources. By the time we consolidate the data and analyze it…it’s out-of-date. That’s where Atlas on Demand comes in…Atlas takes geographically dispersed data from multiple sources and plots it on a map, on demand. Who would use Atlas? Anyone who needs to understand volumes of data that is geographically dispersed and changes over time.

The Menu
The 'Menu Button' is located on the TOP RIGHT section of the ATLAS viewport. To access The Menu, use your mouse to click on it.

Through the Menu, you can access several powerful Atlas features such as FEEDS, LAYERS, BASE MAPS, and MESSAGES. Other features and functions that can be found in The Menu include HELP, FEEDBACK, SETTINGS, and SIGN OUT.

Working With Feeds
Creating A Feed
Some of the fields in the data feed are used to build the map. Other fields allow the user to display location-specific, user-defined data. The data feeds will use six fields to build the map. Three of these fields (highlighted below) are required fields, and the other three fields are optional fields.

The Latitude Field
The Longitude Field
The Name Field

You can only share feeds that belong to you. Feeds that belong to you are distinguishable by the number of options you have access to under the Feeds Tab.

FIELD REQUIRED DESCRIPTION OPTIONS
Latitude YES The latitude in decimal degrees (38.889722) N/A
Longitude YES The longitude in decimal degrees (-77.008889) N/A
Name YES Displayed in the title bar of the info window when a user clicks on a map point. N/A
Symbol NO The shape of the marker (pin). circle, square, triangle. defaults to circle.
status_color NO Specifies the color of this map point. Can be either hex format or a string option (e.g., "#000000" for black or "#FFFFFF" for white). String options: red, orange, blue, black, green. Defaults to blue.
status_name NO Defines the meaning of this map point's color. Displayed in the info window when a user clicks on a map point. Also used to display a legend on the map for aggregation summary. N/A
Optional User-Defined Fields
Additional fields can be included in the data feed as information to be displayed in an info window when a map point is clicked. Atlas simply displays the data included in these fields, with one exception: URLs. The user can include URLs in the data feed, and Atlas will create a clickable button for each URL that open a window to that address.

Feeds can have as many Optional Fields as you like. Optional Field Names, and the corresponding data, are displayed in an info window when a user clicks on a map point. If the field name has multiple words, separate the words with an underscore. Atlas on Demand will replace the underscore with a space when displaying the field name (e.g., "report_date" would be displayed as "Report Date").
Formatting
Field names can also specify the formatting of the data by including a format key in the field name. Format Keys must be all uppercase and appended to the end of the optional field name, separated by a hyphen (e.g., report_date-DATE).

The ADD FEED Button is located on the Top Left corner of the Menu Panel.

Select the file you wish to upload and click OPEN.
Atlas will then open a document window so that you can browse for the file that you want to upload.
