A Feed is the visual representation of data.

Displaying Feeds
Viewing Feeds

To display a feed, toggle the VISIBLE switch on and Atlas will map the data.

Activating a Feed

There are two different kinds of feeds: feeds that you “own” (you’ve uploaded the data) and feeds that someone else has shared with you. Note that you can only share feeds that you own.

Feed Control Hierarchy
Pins

Pins are the visual representation of the data in a FEED, displayed on the map. The map pins are also clickable. Click a pin to display the information available for that pin.

Legends

Atlas creates a legend for each data feed that explains the pin colors and summarizes the data. The 'Legend Buttons' appear at the BOTTOM LEFT corner of the screen whenever a Feed is made visible. Legends stack for each open feed.

Creating Feeds

Creating a data feed is as simple as creating and uploading a CSV file. Atlas Viewer uses fields in the CSV to build the map.

Map-Building Fields

There are six map-building fields that can be included in the CSV. Three of these fields are required and three are optional.

FIELD REQUIRED DESCRIPTION OPTIONS
Latitude YES The latitude in decimal degrees (38.889722) N/A
Longitude YES The longitude in decimal degrees (-77.008889) N/A
Name YES Displayed in the title bar of the info window when a user clicks on a map point. N/A
Symbol NO The shape of the marker (pin). circle, square, triangle. defaults to circle.
status_color NO Specifies the color of this map point. Can be either hex format or a string option (e.g., "#000000" for black or "#FFFFFF" for white). String options: red, orange, blue, black, green. Defaults to blue.
status_name NO Defines the meaning of this map point's color. Displayed in the info window when a user clicks on a map point. Also used to display a legend on the map for aggregation summary. N/A
User-Defined Fields

The CSV can include user-defined fields. Atlas Viewer will display this “metadata” when a user clicks on a map pin.

Clicking on a Pin

Note that field names will be converted to “Title” case when displayed (Atlas Viewer does not, however, automatically reformat the metadata). If the field name has multiple words, separate the words with an underscore. Atlas will replace the underscore with a space when displaying the field name e.g., "date_of_service" would be displayed as "Date of Service").

Formatting

Atlas Viewer displays the metadata (see User-Defined fields) exactly as it’s received; no formatting of the data). Atlas Viewer does, however, allow you to specify formatting of the metadata by attaching a format key to the field name. Format Keys must be all uppercase and appended to the end of the user-defined field name, separated by a hyphen (e.g., report_date-DATE).

FORMAT / KEY INPUT OUTPUT
-DATE 7/8/2014 or 8-Jul-14 7/8/14
-TIME 1:30PM or 13:30 1:30PM
-DATETIME 7/8/14 1:30PM or 7/8/2014 13:30 7/8/14 1:30PM
-DOLLAR 2310 or $2320.00 $2320.00
-PERCENT .05 or 5% 5.0%

Atlas Viewer also gives you the option to create clickable buttons out of URLs. If you append –URL to the end of a user-defined field name and include URLs as the data, Atlas will create a clickable button for each URL that opens a window to that address.

*?* Successfully adding a URL to your CSV makes Atlas display your link as a Button.

Optional Map-Building Fields

The Optional Fields are highlighted.
symbol

The shape of the pins on the map is determined by the “symbol” field. In this example, the pins will be displayed as squares. If the symbol field is left out of the feed, Atlas Viewer will use the default shape for pins (circle).

status_color

Atlas Viewer will color the map pins based on the status_color field. In this example, fire claims will be displayed with a red pin, hail claims will be displayed with a blue pin, liability claims will be displayed with an orange pin, etc. Note that you have unlimited color options. You can use one of the five pre-defined color names (red, blue, green, orange, black), or you can specify the color using hex (e.g., Misc claims will be displayed as magenta in this example).

Sharing Feeds
Share Manager

Share Manager is the easy, secure way to let others view your data. When you share a data feed, the viewers can see the mapped data, but can’t update the data or share it with anyone else. You own the data and you control the data.

Click the “Share” icon next to any of your feeds to manage the users who can view the data.

Opening The Share Manager

For users you’ve already invited, you can see when you invited them and the last time they viewed the feed. You can also revoke their access to the data feed.

Sending An Invitation

You can send an invitation to anyone who has an email address. Click the “Share” icon to open Share Manager and enter the user’s email address:

It’s that easy. If they recipient is a registered Atlas Viewer user, your data feed appears automatically in that user’s list of “invited” feeds. If they aren’t a registered user, the invitation includes instructions for registering with Atlas Viewer.

Revoking Access

You can revoke a user’s access to your data at any time. When you revoke their access, the feed will be removed from their “invited” feeds and they will no longer be able to see the data.

Receiving An Invitation

When a user invites you to view one of their data feeds, you will receive an email from the Atlas Team. If you have already registered with Atlas Viewer, there is no action required on your part. Log in to Atlas and click the FEEDS section of the menu. The shared feed will be in the “invited” section.

If you are not registered, simply click on the link that is in the email (or go to http://atlasviewer.com). Once you register and log in, the shared feed(s) will be in “invited” section of your FEEDS.

Updating Feeds

You can update any of the feeds that you own through the Atlas Viewer MENU. Click the Update icon next to the feed that you want to update.

Updating A Feed

Atlas will then open a document window so that you can browse for the new version of the file. Select the file you wish to upload and click OPEN.

Toggle the VISIBLE switch on and Atlas will map the updated data feed.

Deleting Feeds

Deleting a Feed completely removes that feed from Atlas. The Feed will be deleted from ALL feed lists (including users you’ve invited, as well as your own list of feeds).
